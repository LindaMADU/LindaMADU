### 🚀 Featured Project — ELT Pipeline: dbt + Snowflake Analytics Platform

> End-to-end ELT pipeline ingesting raw data from multiple source systems into Snowflake, transformed and modelled using dbt with a layered architecture, and surfaced via Power BI dashboards for business stakeholders.

<p>
  <img src="https://img.shields.io/badge/Snowflake-29B5E8?style=flat-square&logo=snowflake&logoColor=white" />
  <img src="https://img.shields.io/badge/dbt%20Core-FF694B?style=flat-square&logo=dbt&logoColor=white" />
  <img src="https://img.shields.io/badge/Apache%20Airflow-017CEE?style=flat-square&logo=apacheairflow&logoColor=white" />
  <img src="https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white" />
  <img src="https://img.shields.io/badge/Azure%20Data%20Factory-0078D4?style=flat-square&logo=microsoftazure&logoColor=white" />
  <img src="https://img.shields.io/badge/Power%20BI-F2C811?style=flat-square&logo=powerbi&logoColor=black" />
</p>

**Pipeline Architecture:**

```
Source Systems → Azure Data Factory (Ingest) → Snowflake Raw Layer
     → dbt Staging Models → dbt Intermediate Models → dbt Marts
          → Snowflake Data Warehouse → Power BI Dashboards
```

| Metric | Value |
|--------|-------|
| 🔷 dbt Models | 50+ |
| 🔗 Source Systems | 5 |
| ✅ Pipeline Uptime | 99.8% |
| 🕐 Schedule | Daily |

**Key highlights:**
- Designed a **three-layer dbt architecture** (staging → intermediate → marts) with full data lineage and automated tests
- Orchestrated ingestion and transformation DAGs in **Apache Airflow** with retry logic and Slack alerting
- Leveraged **Snowflake** clustering keys and result caching to reduce query costs by ~40%
- Delivered self-serve **Power BI** dashboards consumed by cross-functional business teams
